## The 5 fidelity statuses the owner tracks

A row is fully faithful only when ALL FIVE of these hold. The table's Status column reports the WEAKEST aspect each row touches; a row can be `identique` on equations yet still gate on a later status.

1. equations_identical  - the governing PDE term (or closure/sign/parameter) matches the paper verbatim.
2. time_algorithm_identical - the time integrator (split, theta-scheme, RK stage count, restart policy) matches the paper.
3. geometry_identical - the spatial domain and BCs are body-fitted to the paper's disc D (radius R), not approximated.
4. backend_validated - the result is reproduced device-clean on the target backend (Kokkos GH200) and/or MPI multi-rank.
5. quantitative_result_reproduced - the numerical growth rate lands on the paper target within the paper's stated deviation.

Per-row Status uses three labels: `identique` (matches the paper for the aspect in that row), `difference_intentionnelle` (a deliberate, documented design choice), `non_verifie` (cannot be confirmed from code or paper, OR confirmable form but unconfirmed numerical outcome).

## The 3 intentional architectural differences (by design, not defects)

These are global engineering choices that the owner accepts up front; individual rows below inherit them.

- FV vs FE: the paper uses a continuous/graph-viscosity finite-element discretization on body-fitted curved quads; ADC is a finite-volume cell-centred scheme. This drives the geometry and fluid-BC rows.
- AMR: ADC carries an adaptive-mesh path (`amr-imex` engine) absent from the paper's uniformly-refined runs. The AMR path further deviates (no CondensedSchur, zero initial momentum, first-order forward-Euler transport).
- Kokkos / MPI: ADC targets a Kokkos device backend and MPI domain decomposition (the paper is CPU FE). The `system-schur` reference path is mono-rank by design; only `amr-imex` runs under MPI.

## Resolved-since-audit note (|Omega|)

The raw Agent-A audit flagged the |Omega| value as `non_verifie`, reading the paper as "omega_c^2 = beta^2 => |Omega| = beta = 1e6". On re-reading the paper this is RESOLVED to `identique`: paper line 278 defines `omega_c := |Omega|_l2` (the cyclotron frequency IS the magnitude of Omega) and line 1082 sets `omega_c := beta^2`. Therefore `|Omega| = beta^2 = 1e12`, which equals `model.py` `omega = beta*beta = 1e12`. The derived-constants and initial-drift rows below carry this correction; the eq (2.6) `1/|Omega|^2` cross product nets a `1/|Omega|` 2D factor and model.py divides by `omega = |Omega| = 1e12`, consistent.
